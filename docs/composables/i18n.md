# i18n Composables

Auto-imported from `app/composables/i18n/`. These composables format values and switch locales based on the active `@nuxtjs/i18n` configuration in `nuxt.config.ts`.

**Prerequisites:** Each locale in `nuxt.config.ts` should define `language`, `dir`, and `currency` (see `LocaleConfig` in `app/types/utils/i18n.ts`).

---

## `useLanguageSwitcher`

Language list and navigation for locale switching.

**When to use:** Header language dropdowns, settings pages, or any UI that lets users change the site language.

**Returns:**

| Name               | Type                    | Description                              |
| ------------------ | ----------------------- | ---------------------------------------- |
| `selectedLocale`   | `ComputedRef<LocaleConfig>` | Current locale with currency metadata |
| `availableLocales` | `ComputedRef<LocaleObject[]>` | All locales except the active one   |
| `switchLanguage`   | `(code) => Promise<void>` | Navigates to the localized route      |

**Example:**

```vue
<script setup lang="ts">
const { selectedLocale, availableLocales, switchLanguage } =
  useLanguageSwitcher();
</script>

<template>
  <UDropdownMenu :items="availableLocales">
    <UButton>{{ selectedLocale.localName }}</UButton>
    <template #item="{ item }">
      <button @click="switchLanguage(item.code)">
        {{ item.localName }}
      </button>
    </template>
  </UDropdownMenu>
</template>
```

**Notes:**

- Uses full page navigation (`window.location.href`) so SSR and route rules stay consistent.
- Requires `currency`, `language`, and optionally `localName` on each locale in `nuxt.config.ts`.

## `useCurrencyFormatter`

Currency formatting driven by the active locale's `currency` config.

**When to use:** Prices, totals, invoices, or any monetary display.

**Methods:**

| Method                  | Description                                           |
| ----------------------- | ----------------------------------------------------- |
| `formatCurrency`        | Standard `Intl` currency string                       |
| `formatCustomCurrency`  | Amount and symbol separately (no decimals)            |
| `formatCurrencyString`  | Symbol positioned per locale (`before` / `after`)     |

**Example:**

```vue
<script setup lang="ts">
const { formatCurrency, formatCurrencyString, formatCustomCurrency } =
  useCurrencyFormatter();

const price = formatCurrency(1250000);
const styled = formatCustomCurrency(1250000);
</script>

<template>
  <p>{{ price }}</p>
  <p>
    <span v-if="styled.symbol">{{ styled.symbol }}</span>
    {{ styled.amount }}
  </p>
  <p>{{ formatCurrencyString(1250000) }}</p>
</template>
```

**Notes:**

- Invalid amounts return `"---"` and log a warning in development.
- Currency code comes from `locale.currency.code` in `nuxt.config.ts`.

## `useNumberFormatter`

Number, percent, and digit conversion helpers.

**When to use:** Stats, counters, forms with Persian/Arabic numerals, or localized percentages.

**Methods:**

| Method                   | Description                                      |
| ------------------------ | ------------------------------------------------ |
| `formatNumber`           | Locale-aware number formatting                   |
| `formatPercent`          | Percentage with configurable fraction digits     |
| `convertToEnglishDigits` | Arabic/Persian digits → ASCII digits             |
| `convertToPersianDigits` | ASCII digits → Persian digits                    |

**Example:**

```vue
<script setup lang="ts">
const { formatNumber, formatPercent, convertToPersianDigits } =
  useNumberFormatter();
</script>

<template>
  <p>{{ formatNumber(1234567.89) }}</p>
  <p>{{ formatPercent(42.5) }}</p>
  <p>{{ convertToPersianDigits("2024") }}</p>
</template>
```

**Notes:**

- Uses `useErrorHandler` for invalid input logging (no toast by default).
- `formatPercent` expects a whole number (e.g. `42.5` → 42.5%).

## `useDateTimeFormatter`

Date and time formatting with i18n reactivity.

**When to use:** Blog posts, timestamps, event dates, or relative "today/yesterday" labels.

**Dependencies:** `@internationalized/date` (included via `@nuxt/ui`).

**Methods:**

| Method                    | Description                                      |
| ------------------------- | ------------------------------------------------ |
| `formatDate`              | Date only                                        |
| `formatTime`              | Time only                                        |
| `formatDateTime`          | Date and time combined                           |
| `formatDateSmartRelative` | Relative label for recent dates, absolute otherwise |

**Supported inputs:** ISO strings (`2024-06-10`, `2024-06-10T14:30:00Z`), `Date`, or timestamp.

**Example:**

```vue
<script setup lang="ts">
const { formatDate, formatDateTime, formatDateSmartRelative } =
  useDateTimeFormatter();

const publishedAt = "2024-06-10T10:00:00Z";
</script>

<template>
  <time :datetime="publishedAt">
    {{ formatDateSmartRelative(publishedAt) }}
  </time>
  <p>{{ formatDateTime(publishedAt) }}</p>
</template>
```

**Notes:**

- Empty input returns `""`; invalid dates return `"---"`.
- `formatDateSmartRelative` uses `Intl.RelativeTimeFormat` (locale-aware) for recent dates within the threshold (default 7 days).
- UTC day boundaries avoid DST calculation issues.
